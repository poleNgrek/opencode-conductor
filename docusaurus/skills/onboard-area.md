# onboard-area

Read-only skill for orienting an agent (or human) into an area's architecture, conventions, and ownership.

## What it does

- Reads area-level `AGENTS.md`.
- Summarizes architecture, dependencies, and verification gates.
- Surfaces a quick "first 30 minutes" reading list.

## Recommended permission

`allow` — non-mutating.

## When to use

- Onboarding into an unfamiliar area.
- Before a multi-area review pass.
