`/project-help-docs` and `skills/help-docs-author` provide a deterministic, code-first workflow for generating end-user help-center documentation.

Default workflow:

1. Discovery
2. Code-reading
3. Plan
4. Generation
5. Audit

Safety defaults:

- Output containment is enforced.
- In-repo output is refused unless explicitly allowed.
- Frontmatter and vocabulary audit are enabled by default.
