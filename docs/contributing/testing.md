Testing guidance is maintained in `TEST_PLAN.md`, including scenario-based checks for Plan 1/2/3 command families.

Recommended verification order for doc or workflow changes:

1. Run through relevant `TEST_PLAN.md` sections.
2. Execute `git status` and inspect changed files.
3. Confirm vendor-neutrality for upstream changes.
4. Validate links/frontmatter when adding docs pages.
